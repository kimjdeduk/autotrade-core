# 외부 연결 목록

이 문서는 현재 프로젝트에서 사용할 수 있거나
앞으로 연결할 가능성이 있는 외부 서비스, 데이터 소스, 플랫폼을 정리하는 문서다.

## 1. 문서 목적
외부 연결 목록을 정리하는 이유는 아래와 같다.

- 어떤 외부 자원을 쓰는지 한눈에 보기 위해
- collectors, config, skills 설계를 쉽게 하기 위해
- API 연결과 운영 연결을 구분하기 위해
- 나중에 경로와 설정이 꼬이지 않게 하기 위해

## 2. 분류 기준
외부 연결은 아래 4가지로 나눈다.

1. 데이터/API 연결
2. 개발/버전관리 연결
3. 운영/알림 연결
4. 문서/협업 연결

## 3. 데이터/API 연결
현재 또는 향후 사용할 수 있는 데이터/API 연결 목록

- KIS
  - 용도: 시세 조회, 잔고 조회, 주문 관련 기능
  - 분류: 핵심 데이터/API 연결

- KRX
  - 용도: 한국거래소 데이터 조회
  - 분류: 보조 데이터/API 연결

- DART
  - 용도: 공시 정보 조회
  - 분류: 공시/기초정보 연결

- 네이버 증권
  - 용도: 보조 참고 정보 확인
  - 분류: 참고 데이터 연결

- CSV 파일
  - 용도: 백테스트 데이터, 수동 저장 데이터
  - 분류: 로컬 데이터 연결

## 4. 개발/버전관리 연결
- Git
  - 용도: 버전관리, 롤백, 변경 이력 관리
  - 분류: 필수 개발 연결

- GitHub
  - 용도: 원격 저장소, 코드 백업, 협업 반영
  - 분류: 원격 개발 연결

- VS Code
  - 용도: 외부 개발 환경
  - 분류: 개발 도구

- Python 가상환경
  - 용도: 프로젝트 의존성 분리
  - 분류: 개발 실행 환경

## 5. 운영/알림 연결
- OpenClaw
  - 용도: 실행 지시, 검수, 요약, 보고
  - 분류: 운영 중심 연결

- Telegram
  - 용도: 결과 알림, 보고 전송, 운영 메시지 수신
  - 분류: 알림 연결

## 6. 문서/협업 연결
- Google Drive
  - 용도: 문서, 스프레드시트, 참고자료 저장
  - 분류: 문서 연결

- Slack
  - 용도: 운영 메시지, 협업 기록, 공지 확인
  - 분류: 협업 연결

- 프로젝트 소스 문서
  - 용도: 기준 문서 저장
  - 분류: 내부 지식 연결

### OpenClaw에서 주로 쓰는 연결

### 양쪽 모두 관련 있는 연결
- GitHub

- OpenClaw
- VS Code

## 8. 관리 원칙
- 데이터 수집 연결은 collectors와 config에서 관리한다.
- API 키, 토큰, 계정 정보는 코드에 직접 박지 않는다.
- 민감한 값은 환경변수나 별도 설정 파일로 관리한다.
- 외부 연결 추가 시 반드시 이 문서에 기록한다.
- 연결이 중단되면 어떤 기능에 영향이 있는지 함께 기록한다.
