# 결정사항 및 진행상황 누적 문서

이 문서는 프로젝트에서 확정된 결정사항과
현재까지의 진행상황을 계속 누적해서 기록하는 문서다.

## 1. 문서 목적
이 문서를 두는 이유는 아래와 같다.

- 길어진 대화에서 중요한 결정사항을 잊지 않기 위해
- 현재 어디까지 진행됐는지 한눈에 보기 위해
- 역할, 구조, 규칙 변경 내역을 남기기 위해
- 다음 작업 우선순위를 분명하게 하기 위해

## 2. 기본 사용 원칙
- 중요한 결정이 나오면 바로 이 문서에 반영한다.
- 구조 변경이 있으면 변경 이유까지 함께 적는다.
- 역할 변경, 파일 규칙 변경, 운영 규칙 변경은 반드시 기록한다.
- 긴 채팅 후 새 채팅으로 넘어갈 때 이 문서를 먼저 확인한다.

## 3. 현재까지 확정된 핵심 결정사항

### 3-1. 프로젝트 방향
- 최종 목표는 자동매매 시스템 구축이다.
- 현재 우선 목표는 시스템 구축이다.
- 지금은 자동매매 실행보다
  자동매매를 만들 수 있는 구조를 먼저 확립하는 단계다.

### 3-2. 역할 구조
- 시스템팀: 아사, 아싸
- 자동매매팀: 아자, 아짜
- 최종 검수 및 최종 판단: 아짜
- 사자는 현재 제외한다.

### 3-3. 시스템 경계
- autotrade-core는 OpenClaw 밖에 둔다.
- autotrade-core는 실제 프로그램 본체를 개발하는 곳이다.
- OpenClaw는 실행, 검수, 요약, 보고 중심으로 사용한다.
- 밖에서 만들고 안에서는 돌리고 검수만 한다.

### 3-4. 구조 원칙
- 무거운 로직은 OpenClaw 안에 넣지 않는다.
- 계산은 밖에서 하고 정리는 안에서 한다.
- 결과 파일은 outputs 폴더 기준으로 저장한다.
- 한 파일 안에 여러 역할을 섞지 않는다.

### 3-5. 작업 원칙
- 큰 작업을 한 번에 하지 않는다.
- 작은 기능 단위로 나눠서 만든다.
- 시스템 구축과 자동매매 전략 작업을 분리한다.
- 오류 수정은 로그를 보고 따로 처리한다.
- 실주문 연결은 마지막 단계에서 검토한다.

## 4. 현재까지 준비된 기준 문서
현재 프로젝트 소스에 아래 문서들을 준비한다.

1. 역할 구조
2. 프로젝트 목표
3. 시스템 경계
4. 표준 폴더 구조
5. 결정사항 요약
6. 파일명 규칙
7. 운영 규칙
8. 기존 자산 분류
9. 외부 연결 목록
10. 결정사항 및 진행상황 누적 문서

## 5. 현재 진행상황
현재까지 진행상황은 아래와 같다.

### 완료
- 프로젝트 방향 확정
- 역할 구조 초안 확정
- 외부 개발 / 내부 운영 분리 원칙 확정
- 표준 폴더 구조 초안 정리
- 파일명 규칙 초안 정리
- 운영 규칙 초안 정리
- 기존 자산 분류 기준 정리
- 외부 연결 목록 정리
- 프로젝트 소스용 기준 문서 제작 진행

### 진행 중
- 프로젝트 소스 정리
- 기존 자산 실제 분류
- OpenClaw 내부 재활용 대상 정리
- autotrade-core 실제 초기 폴더 구성 준비

### 아직 하지 않음
- 실전 자동매매 로직 본체 개발
- 실주문 연결
- 리스크 엔진 구현
- 자동매매 전체 통합 실행 구조 완성

## 9. 사용 방법
- 새 채팅을 시작할 때 이 문서를 먼저 확인한다.
- 현재 상태를 잊었을 때 이 문서를 기준으로 다시 시작한다.
- 중요한 결정을 말로만 하지 말고 이 문서에 반영한다.
- 프로젝트가 길어질수록 이 문서의 중요도가 높아진다.
