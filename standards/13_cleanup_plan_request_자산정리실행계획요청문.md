# 자산 정리 실행 계획 요청문

이 문서는 기존 자산 상세 분석이 끝난 뒤
실제 정리 작업에 들어가기 전에 정리 실행 계획을 만들기 위한 요청문이다.

## 1. 문서 목적
이 문서를 두는 이유는 아래와 같다.

- 자료 수집 단계 다음에 무엇을 해야 하는지 분명하게 하기 위해
- 실제 삭제, 이동, 통합 전에 안전한 실행 계획을 먼저 세우기 위해
- 큰 정리 작업을 작은 단계로 나누기 위해
- OpenClaw 안정성을 해치지 않고 구조 정리를 진행하기 위해

## 2. 현재 전제
현재 분석 결과상 아래 내용이 이미 1차 파악되었다.

- autotrade-strategy-lab = 위험도 높음, 분리 권장
- tools 디렉토리 = 중복, 통합 권장
- invest-ops SKILL = 수정 후 유지 권장
- config/weights/long_term_v1.yaml = 수정 후 유지 권장
- data/raw 디렉토리 구조 = 수정 후 유지 권장
- OpenClaw에 남길 핵심 자산과 autotrade-core로 옮길 핵심 자산도 1차 정리됨

즉 지금 단계는
자료 수집 다음 단계로서
실제 수정 전에 정리 실행 계획을 세워야 하는 단계다.

## 3. 아사에게 요청할 작업 목적
이번 요청의 목적은
실제 삭제, 이동, 수정, 통합을 바로 하는 것이 아니라
안전한 정리 순서와 작업 단위를 먼저 확정하는 것이다.
