# 기존 자산 실제 목록 양식

이 문서는 현재 시스템 안에 이미 존재하는 스킬, 코드, 문서, 설정, 템플릿을
실제로 하나씩 적어 넣기 위한 기록 양식 문서다.

## 1. 문서 목적
이 문서를 두는 이유는 아래와 같다.

- 지금 가지고 있는 자산을 빠짐없이 모으기 위해
- 재활용 가능한 것을 찾기 위해
- 앞으로 OpenClaw와 autotrade-core로 나눌 기준을 잡기 위해
- 기존 자산 분류 문서와 연결해 실제 작업을 하기 위해

## 2. 사용 방법
- 지금 가지고 있는 자산을 보이는 대로 하나씩 적는다.
- 처음에는 상세 내용이 없어도 된다.
- 이름, 위치, 용도만 먼저 적어도 된다.
- 나중에 분류 항목을 추가로 채운다.

## 3. 작성 기본 원칙
- 자산 1개당 1개 항목으로 기록한다.
- 이름이 비슷해도 별개 파일이면 따로 적는다.
- 코드, 스킬, 문서, 설정, 템플릿을 섞지 말고 유형을 적는다.
- 정확하지 않으면 추정이라고 표시한다.
- 모르는 것은 빈칸으로 두고 나중에 보완한다.

## 4. 자산 기록 양식

### 기본 양식
- 이름:
- 유형:
  - 스킬
  - 코드
  - 문서
  - 설정
  - 템플릿
  - 기타
- 현재 위치:
- 현재 용도:
- 관련 팀:
  - 시스템팀
  - 자동매매팀
  - 공용
- 현재 상태:
  - 사용 중
  - 일부 사용 중
  - 미사용
  - 불명
- 분류:
  - 그대로 사용
  - 수정 후 사용
  - 분리 또는 폐기 검토
  - 미정
- 비고:

## 5. 기록 예시

### 예시 1
- 이름: 로그 요약 스킬
- 유형: 스킬
- 현재 위치: OpenClaw skills 내부
- 현재 용도: 실행 로그 요약
- 관련 팀: 시스템팀
- 현재 상태: 사용 중
- 분류: 그대로 사용
- 비고: 결과 요약 및 보고용

### 예시 2
- 이름: 백테스트 실행 파일
- 유형: 코드
- 현재 위치: 기존 백테스트 폴더
- 현재 용도: 전략 성능 검증
- 관련 팀: 시스템팀
- 현재 상태: 일부 사용 중
- 분류: 수정 후 사용
- 비고: autotrade-core로 이동 검토 필요

### 예시 3
- 이름: 올인원 자동매매 스킬
- 유형: 스킬
- 현재 위치: OpenClaw 내부
- 현재 용도: 수집, 계산, 보고까지 한 번에 처리
- 관련 팀: 자동매매팀
- 현재 상태: 사용 중
- 분류: 분리 또는 폐기 검토
- 비고: 역할이 너무 많이 섞여 있음

## 6. 권장 분류 순서
기존 자산을 정리할 때 아래 순서로 적는 것을 권장한다.

1. 운영/알림 관련 자산
2. 로그/검수 관련 자산
3. 전략/백테스트 관련 자산
4. 데이터 수집 관련 자산
5. 문서/규칙 관련 자산
6. 설정/연결 관련 자산
