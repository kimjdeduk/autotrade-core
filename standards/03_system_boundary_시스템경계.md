# 시스템 경계

이 프로젝트의 핵심 원칙은 외부 개발과 내부 운영의 분리다.

## 1. autotrade-core
autotrade-core는 OpenClaw 밖에 두는 외부 개발 프로젝트다.

### autotrade-core에서 하는 일
- 전략 코드 작성
- 데이터 수집 코드 작성
- 백테스트 코드 작성
- 실시간 감시 코드 작성
- 로그 저장 코드 작성
- 리포트 생성 코드 작성
- 테스트 코드 작성

즉 autotrade-core는 실제 프로그램 본체를 만드는 개발실이다.

## 2. OpenClaw
OpenClaw는 프로그램 본체를 직접 만드는 곳이 아니다.

### OpenClaw에서 하는 일
- 실행 명령 호출
- 실행 결과 읽기
- 결과 요약
- 로그 검수
- 이상 상황 보고
- 텔레그램 알림 전송
- 역할별 검토
- 보고서 정리

즉 OpenClaw는 운영실, 관제실, 검수실이다.

## 3. 핵심 원칙
- 밖에서 만들고 안에서는 돌리고 검수만 한다.
- 무거운 로직은 autotrade-core에 둔다.
- OpenClaw에는 실행, 확인, 요약, 보고 규칙만 둔다.
- 계산은 밖에서 하고 정리는 안에서 한다.
