# 역할 구조

현재 이 프로젝트의 역할 구조는 아래와 같다.

## 1. 시스템팀
- 아사
- 아싸

### 시스템팀 담당 업무
- 시스템 관리
- 구조 설계
- 폴더 구조 적용
- 파일 구조 정리
- 분석
- 백테스트
- 오류 원인 파악
- 운영 안정화

## 2. 자동매매팀
- 아자
- 아짜

### 자동매매팀 담당 업무
- 자동매매 전략 작업
- 매매 로직 검토
- 결과 해석
- 리스크 점검
- 검수
- 최종 판단

## 3. 최종 판단
- 아짜가 최종 검수 및 최종 판단 역할을 담당한다.

## 4. 현재 원칙
- 사자는 현재 제외한다.
- 시스템 구축과 자동매매 작업은 분리한다.
- 시스템팀은 구축과 검증 중심으로 움직인다.
- 자동매매팀은 전략과 실행 검토 중심으로 움직인다.
