# 시스템 구축 실행 계획

이 문서는 현재 프로젝트에서 기준 문서 1번부터 10번까지를 실제 시스템 구축에 반영하기 위한 실행 계획 문서다.

## 1. 문서 목적
- 기준 문서를 실제 작업 순서로 연결하기 위해
- 지금 무엇을 먼저 해야 하는지 분명하게 하기 위해
- 시스템 구축을 문서 단계에서 실제 적용 단계로 넘기기 위해
- 큰 작업을 작게 쪼개어 진행하기 위해

## 2. 현재 단계 정의
현재 이 프로젝트는 자동매매 실행 단계가 아니다.

현재 단계는 아래 3가지를 먼저 확립하는 단계다.
1. OpenClaw 내부 LLM 구조 확립
2. 시스템 구조 확립
3. 파일 구조 확립

즉 지금은 자동매매를 바로 돌리는 단계가 아니라 자동매매 시스템을 안정적으로 만들 수 있는 기반을 세우는 단계다.

## 3. 전체 실행 순서
1. 프로젝트 생성
2. 기준 문서 1번~10번 프로젝트 소스에 반영
3. 프로젝트 원칙 확인
4. 기존 자산 목록 수집
5. 기존 자산 분류
6. 실제 파일 구조 확인
7. autotrade-core 실제 생성
8. OpenClaw 내부 재활용 구조 정리
9. 첫 기능 1개 선정
10. 파일 단위 제작 시작
11. 단독 실행 테스트
12. 결과 저장 확인
13. OpenClaw 연결 준비

## 4. 단계별 실행 계획

### 1단계. 프로젝트 생성
- 새 프로젝트 1개 생성
- 프로젝트 이름 확정
- 이 프로젝트를 시스템 구축 전용 프로젝트로 정의

### 2단계. 기준 문서 1번~10번 반영
반영 문서:
1. 역할 구조
2. 프로젝트 목표
3. 시스템 경계
4. 표준 폴더 구조
5. 결정사항 요약
6. 파일명 규칙
7. 운영 규칙
8. 기존 자산 분류
9. 외부 연결 목록
10. 결정사항 및 진행상황 누적

### 3단계. 프로젝트 원칙 확인
- 이 프로젝트는 자동매매 실행 프로젝트가 아니라 시스템 구축 프로젝트라고 확정
- autotrade-core는 OpenClaw 밖에 둔다는 원칙 고정
- OpenClaw는 실행, 검수, 요약, 보고 중심이라는 원칙 고정

### 4단계. 기존 자산 목록 수집
- 현재 가지고 있는 스킬, 코드, 문서, 설정, 템플릿을 이름 기준으로 먼저 모은다.

### 5단계. 기존 자산 분류
분류 기준:
1. 그대로 사용
2. 수정 후 사용
3. 분리 또는 폐기 검토

### 6단계. 실제 파일 구조 확인
- 현재 OpenClaw 내부 구조 확인
- 현재 작업 폴더 구조 확인
- 앞으로 만들 autotrade-core 위치 확인

### 7단계. autotrade-core 실제 생성
기본 폴더:
- config
- data
- collectors
- strategies
- backtests
- runners
- reports
- logs
- tests
- outputs

### 8단계. OpenClaw 내부 재활용 구조 정리
OpenClaw에 남길 것 예시:
- 실행 지시
- 결과 요약
- 로그 검수
- 텔레그램 전송
- 보고서 정리

밖으로 뺄 것 예시:
- 데이터 수집 본체
- 전략 계산 본체
- 백테스트 본체
- 리스크 계산 본체

### 9단계. 첫 기능 1개 선정
추천 예시:
- 결과 파일 저장 구조 만들기
- 빈 today_signals.json 생성기 만들기
- 백테스트 실행 틀 만들기
- 로그 파일 생성 구조 만들기

### 10단계. 파일 단위 제작 시작
- 필요한 파일을 하나씩 만든다.
- 한 번에 전체 시스템을 만들지 않는다.

### 11단계. 단독 실행 테스트
- OpenClaw 없이 직접 실행한다.
- 오류 여부를 먼저 확인한다.

### 12단계. 결과 저장 확인
- 실행 결과가 outputs에 저장되는지 확인한다.

### 13단계. OpenClaw 연결 준비
- OpenClaw에서 실행할 수 있는 지시 구조를 만든다.
- 결과 파일을 읽고 요약하는 흐름을 정리한다.

## 5. 작업 원칙
- 큰 작업을 한 번에 하지 않는다.
- 한 번에 하나의 기능만 만든다.
- 구조와 전략을 섞지 않는다.
- 실주문 연결은 마지막에 검토한다.
- 결과 파일 저장 원칙을 지킨다.

## 6. 지금 바로 해야 할 일
1. 프로젝트 1개 생성
2. 문서 1번부터 10번까지 프로젝트 소스에 각각 반영
3. 기준 문서가 잘 들어갔는지 확인
4. 기존 자산 이름 목록 만들기
5. 기존 자산 분류 시작

## 7. 한 줄 요약
이 문서는 문서로만 정리된 기준을 실제 시스템 구축 순서로 바꾸어 실행하기 위한 작업 계획 문서다.
