# 사자(Manus) ↔ 아사(OpenClaw) 직접 통신 자동화 방안

**작성자**: 사자 (Manus AI)
**작성일**: 2026년 4월 13일

현재 Commander가 사자에게 지시를 내리면, 사자가 GitHub에 지시문을 올리고 Commander가 다시 아사에게 "pull 받고 실행해"라고 텔레그램으로 명령하는 구조입니다. 이 중간 단계를 생략하고 **사자가 아사에게 직접 지시를 전달하고 실행까지 시키는 자동화 방안**을 조사한 결과입니다.

## 1. 현재 통신 구조의 한계 분석

텔레그램 봇 API의 구조적 특성상, 사자(Manus)가 텔레그램을 통해 아사(OpenClaw)에게 직접 "명령"을 내리는 것은 불가능합니다.

*   **이유 1**: 텔레그램 봇(아사)은 사용자(Commander)의 메시지만 수신할 수 있으며, 다른 봇이나 외부 시스템이 봇에게 직접 메시지를 보낼 수 없습니다.
*   **이유 2**: 사자가 아사 봇의 토큰을 이용해 Commander의 채팅방에 메시지를 보낼 수는 있지만(테스트 완료), 이는 아사가 "수신"하는 것이 아니라 아사가 Commander에게 "발신"하는 형태가 되므로 아사의 실행 트리거로 작동하지 않습니다.

따라서 텔레그램을 우회하는 **GitHub 기반 자동화** 또는 **직접 API 호출** 방식이 필요합니다.

## 2. 실현 가능한 자동화 방안 3가지

### 방안 A: GitHub Polling (가장 추천, 즉시 적용 가능)
아사(WSL 환경) 쪽에 1분 주기로 GitHub `instructions/` 폴더를 감시하는 스크립트를 백그라운드로 띄워두는 방식입니다.

*   **작동 방식**:
    1. Commander가 사자에게 지시.
    2. 사자가 `instructions/` 폴더에 새 지시문(`.md`)을 push.
    3. 아사의 백그라운드 스크립트가 1분마다 `git pull`을 수행하여 새 파일이 감지되면 자동으로 읽고 실행.
    4. 실행 완료 후 아사가 결과를 push하고 알림봇으로 Commander에게 보고.
*   **장점**: 방화벽이나 포트포워딩 설정이 필요 없으며, 현재 구조에서 아사 쪽에 스크립트 하나만 추가하면 바로 작동합니다.
*   **단점**: 최대 1분의 지연 시간이 발생할 수 있습니다.

### 방안 B: OpenClaw Gateway 직접 호출 (가장 빠름, 설정 필요)
아사(OpenClaw)는 현재 포트 18789에서 systemd 서비스로 동작 중입니다. 이 포트를 외부로 열어 사자가 직접 HTTP 요청을 보내는 방식입니다.

*   **작동 방식**:
    1. Commander가 사자에게 지시.
    2. 사자가 아사의 공인 IP(또는 도메인)와 포트(18789)로 직접 API 요청(POST) 전송.
    3. 아사가 즉시 요청을 받아 실행.
*   **장점**: 지연 시간 없이 즉각적으로 실행됩니다.
*   **단점**: Hostinger VPS의 방화벽 설정 변경 및 포트포워딩이 필요하며, 보안을 위해 API 키 인증 로직을 추가해야 합니다.

### 방안 C: GitHub Webhook + 로컬 릴레이
GitHub 저장소에 Webhook을 설정하여, 사자가 push할 때마다 아사 쪽으로 신호를 보내는 방식입니다.

*   **작동 방식**: 사자 push → GitHub Webhook 발생 → 아사 수신 → 자동 pull 및 실행.
*   **장점**: Polling(방안 A)처럼 불필요한 트래픽 낭비가 없고 즉각적입니다.
*   **단점**: 아사(WSL)가 외부에서 Webhook을 수신할 수 있도록 공인 IP/포트 개방 또는 ngrok 같은 터널링 서비스가 필요합니다.

## 3. Commander를 위한 제안 및 다음 단계

가장 안전하고 빠르게 적용할 수 있는 방법은 **방안 A (GitHub Polling)**입니다. Commander의 개입 없이 사자와 아사가 GitHub을 매개로 완벽하게 동기화될 수 있습니다.

**적용을 원하신다면 다음 단계를 진행하겠습니다:**

1.  사자가 아사에게 **"GitHub `instructions/` 폴더를 1분 주기로 감시하고, 새 파일이 들어오면 자동 실행하는 백그라운드 스크립트(`auto_pull_runner.py`)를 작성하여 systemd에 등록하라"**는 지시문을 작성하여 push합니다.
2.  Commander께서 마지막으로 한 번만 아사에게 "pull 받고 실행해"라고 명령하시면, 이후부터는 사자가 지시문을 올리는 즉시 아사가 알아서 실행하는 완전 자동화 루프가 완성됩니다.
