# 계획 문서 vs 실제 시스템 현황 차이점 분석 (Gap Analysis)

**작성자**: 사자 (Manus AI)
**작성일**: 2026년 4월 13일

이 문서는 Commander가 제공한 00~13번 기준 문서(계획)와 현재 GitHub 저장소(`autotrade-core`)의 실제 구현 상태 간의 차이점(Gap)을 분석한 결과입니다. 이 분석을 바탕으로 아사(OpenClaw)에게 실제 현황을 확인하고 정리를 지시할 계획입니다.

## 1. 핵심 원칙 요약 (계획 문서 기준)

00~13번 문서에서 강조하는 가장 중요한 원칙은 다음과 같습니다.

1.  **외부 개발과 내부 운영의 분리**: 무거운 계산, 데이터 수집, 전략 백테스트 본체는 `autotrade-core`에 두고, OpenClaw는 실행 지시, 결과 읽기, 요약, 보고만 담당합니다 [1].
2.  **역할 분리 (폴더 구조)**: 한 파일에 여러 역할을 섞지 않습니다. 수집은 `collectors/`, 계산은 `strategies/`, 실행은 `runners/`, 정리는 `reports/`, 결과 저장은 `outputs/`에서 각각 담당합니다 [2].
3.  **결과 파일 우선 원칙**: 실행 결과는 화면 출력으로 끝내지 않고 반드시 `outputs/` 폴더에 파일(JSON, MD 등)로 저장하며, OpenClaw는 이 파일을 읽어서 보고합니다 [3].
4.  **작업 순서 고정**: 구조 설계 → 파일 생성 → 단독 실행 테스트 → 오류 수정 → 결과 저장 확인 → 보고 형식 정리 → OpenClaw 연결 순서로 진행합니다 [3].
5.  **기존 자산 재활용**: 기존 스킬과 코드는 "그대로 사용", "수정 후 사용", "분리/폐기 검토"로 분류하여 점진적으로 `autotrade-core`로 이관합니다 [4].

## 2. 계획 vs 실제 구현 차이점 (Gap)

현재 GitHub 저장소의 코드를 분석한 결과, 계획 문서의 원칙이 아직 완전히 적용되지 않은 부분들이 발견되었습니다.

### 2.1. 폴더 구조의 부재
*   **계획**: `strategies/`, `runners/`, `reports/`, `outputs/` 등의 세분화된 폴더 구조를 요구합니다 [2].
*   **실제**: 현재 저장소에는 `src/collectors/`, `src/utils/`, `tests/` 폴더만 존재하며, 전략, 실행, 보고, 출력 폴더는 아직 생성되지 않았습니다.

### 2.2. 역할 혼재 (테스트 스크립트)
*   **계획**: 실행은 `runners/`에서, 결과 저장은 `outputs/`에서, 알림은 `reports/` 또는 OpenClaw에서 분리하여 처리해야 합니다 [2].
*   **실제**: `tests/test_ohlcv_collector.py` 파일을 보면, 데이터 수집(`OHLCVCollector`), CSV 파일 저장(`data/raw/`), 화면 출력, 텔레그램 알림 전송(`TelegramNotifier`)이 하나의 테스트 스크립트 안에서 모두 이루어지고 있습니다. 이는 "한 파일에 여러 역할을 섞지 않는다"는 원칙에 위배됩니다.

### 2.3. 결과 저장 위치 불일치
*   **계획**: 실행 결과는 `outputs/` 폴더에 저장해야 합니다 [2].
*   **실제**: 현재 데이터 수집 결과는 `data/raw/` 폴더에 CSV 형태로 저장되고 있습니다. (이는 원천 데이터 저장용으로는 맞으나, 최종 결과물 저장소인 `outputs/`의 활용이 아직 없습니다.)

### 2.4. 기존 자산 정리 미완료
*   **계획**: 13번 문서(자산 정리 실행 계획 요청문)를 통해 아사에게 기존 자산(예: `autotrade-strategy-lab`, `tools` 디렉토리, `invest-ops` 스킬 등)의 정리 계획을 세우도록 지시했습니다 [5].
*   **실제**: 해당 정리 작업이 완료되어 `autotrade-core`로 이관되었는지, 아니면 아직 OpenClaw 쪽에 남아있는지 GitHub 저장소만으로는 확인이 불가능합니다.

## 3. 아사(OpenClaw) 확인 필요 사항

위의 차이점을 바탕으로, 아사에게 다음 사항들을 확인해야 합니다.

1.  **실제 폴더 구조 현황**: 아사의 로컬(WSL) 환경에 `strategies/`, `runners/`, `outputs/` 등의 폴더가 생성되어 있는지 확인.
2.  **기존 자산 정리 진행 상태**: 13번 문서의 지시사항(자산 정리 실행 계획)이 어디까지 진행되었는지 확인.
3.  **역할 분리 적용 계획**: 현재 `tests/test_ohlcv_collector.py`처럼 섞여 있는 로직을 언제 `runners/`와 `outputs/` 구조로 분리할 것인지 확인.
