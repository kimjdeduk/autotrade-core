# autotrade-core 시스템 종합 개요

이 문서는 Commander(김재덕)의 자동매매 시스템 프로젝트(autotrade-core)의 전반적인 시스템 흐름, 환경 구조, 운영 원칙 및 과거 진행 내용을 종합적으로 정리한 문서입니다. 사자(Manus)는 프로젝트의 전체적인 맥락을 파악하고 향후 작업을 계획할 때 이 문서를 참조해야 합니다.

## 1. 환경 구조 및 아키텍처

본 시스템은 Hostinger VPS의 Windows 11 기반 WSL(Ubuntu) 환경에서 구동됩니다. 핵심 컴포넌트인 OpenClaw Gateway는 systemd 서비스로 동작하며, 텔레그램 봇을 통해 사용자와 상호작용합니다.

### 1.1. 주요 텔레그램 봇

시스템 운영에 필수적인 두 개의 텔레그램 봇이 연동되어 있습니다.

| 봇 이름 | 봇 ID | 역할 |
|---|---|---|
| 아사 | @assa_openclaw_bot | OpenClaw 메인 인터페이스, 작업 지시 수신 및 실행 |
| 알림 | @Alarm1072_bot | 시스템 이벤트, 에러, 작업 완료 등의 자동 알림 전송 |

### 1.2. 협업 구조 (사자 ↔ 아사)

사자(Manus)는 AI 참모장으로서 전체 프로젝트를 기획하고 지시하며, 아사(OpenClaw)는 실무자로서 코드를 작성하고 실행합니다. 이들의 협업은 GitHub 저장소를 통해 이루어집니다.

1. **지시 하달:** 사자가 `instructions/` 폴더에 마크다운 형식(`YYYYMMDD_순번_작업명.md`)의 지시문을 작성하여 GitHub에 push합니다.
2. **실행 명령:** Commander가 텔레그램을 통해 아사에게 "pull 받고 실행해"라고 명령합니다.
3. **작업 수행:** 아사는 GitHub에서 지시문을 pull하여 읽고, 코드를 작성 및 실행합니다.
4. **결과 보고:** 아사는 작업 완료 후 결과를 GitHub에 push하고, 지시문 파일 상단에 `상태: 완료`를 명시합니다. 알림봇을 통해 완료 메시지도 전송합니다.
5. **검수:** 사자는 GitHub에서 결과 코드를 직접 pull하여 검수하고, 텔레그램 Bot API를 통해 알림 메시지를 모니터링합니다.

### 1.3. 아사 내부 협업 프로세스

아사는 단일 모델(DeepSeek Chat/V3)을 사용하지만, 프롬프트 시뮬레이션을 통해 5단계의 페르소나 협업 프로세스를 거칩니다.

1. **아사 (초안):** 요구사항 분석 및 초기 코드 초안 작성
2. **아싸 (추론):** 논리적 오류 검증 및 개선 방안 도출
3. **아자 (정리):** 코드 리팩토링 및 주석 추가
4. **아짜 (검수):** 요구사항 충족 여부 최종 확인
5. **사자 (최종 승인):** 아짜의 검수를 통과한 코드를 사자가 최종 승인 및 GitHub 반영

## 2. 운영 원칙

안전하고 효율적인 시스템 운영을 위해 다음의 원칙을 엄격히 준수합니다.

*   **보안 철저:** API 키, 토큰 등 민감 정보는 절대 코드에 하드코딩하지 않으며, 반드시 `.env` 파일로 관리합니다. 로그나 알림 메시지에도 실제 비밀값이 출력되지 않도록 주의합니다.
*   **단계적 접근:** 한 번에 여러 단계를 묶어서 진행하지 않고, `설정 → 확인 → 테스트 → 저장 → 알림 → 커밋`의 순서로 단계별로 진행합니다.
*   **버전 관리 통제:** `.env` 파일 및 로컬 데이터 파일(`data/raw/`, `data/processed/`, `logs/`)은 `.gitignore`에 포함하여 GitHub 커밋을 차단합니다.
*   **사용자 승인 필수:** GitHub push 등 시스템 상태를 변경하는 주요 작업은 반드시 Commander의 승인 하에 진행합니다.

## 3. 프로젝트 진행 상황 (과거 내용)

현재까지 인프라 구축 및 데이터 수집 파이프라인(Phase 1 ~ Phase 2)이 완료되었으며, 전략 엔진 개발(Phase 3)을 앞두고 있습니다.

### 3.1. 완료된 마일스톤

*   **Phase 1 (기본 인프라):** 프로젝트 디렉토리 구조, 가상 환경, 의존성 관리 체계 및 GitHub 저장소 연동 완료.
*   **Phase 1.5 (자동 알림):** 텔레그램 알림봇(@Alarm1072_bot) 연동 및 `notifier.py` 모듈 구현 완료.
*   **Phase 2 (데이터 수집):** 한국투자증권(KIS) OpenAPI 연동(`kis_client.py`) 및 삼성전자 일별 OHLCV 데이터 수집기(`ohlcv_collector.py`) 구현 완료.
*   **자동화 연동 테스트:** 사자와 아사 간의 GitHub 기반 지시문 전달 및 실행 체계 테스트 성공.

### 3.2. 향후 계획

*   **Phase 3 (전략 엔진 개발):** 수집된 OHLCV 데이터를 기반으로 매매 신호를 생성하는 전략 모듈 구현 및 백테스트 시스템 구축.
*   **Phase 4 (주문 실행기 개발):** KIS API를 활용한 실제 매수/매도 주문 실행 모듈 및 위험 관리 모듈 연동.
*   **Phase 5 (시스템 통합 및 실전 테스트):** 전체 모듈 통합 및 모의투자 환경에서의 실전 테스트.
*   **Phase 6 (실거래 전환 및 모니터링):** 실계좌 연동, 실시간 모니터링 대시보드 구축 및 지속적인 전략 최적화.
