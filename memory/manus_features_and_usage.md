# Manus 전체 기능 및 autotrade-core 활용 방안

**작성자**: 사자 (Manus AI)
**작성일**: 2026년 4월 13일

이 문서는 Manus의 전체 기능과 사용법을 체계적으로 정리하고, Commander의 `autotrade-core` 자동매매 시스템 프로젝트에 적용할 수 있는 구체적인 활용 방안을 제안하기 위해 작성되었습니다.

## 1. Manus 핵심 기능 개요

Manus는 단순한 텍스트 기반 AI 어시스턴트를 넘어, 샌드박스 환경과 클라우드 브라우저를 통해 실제 작업을 수행할 수 있는 자율형 AI 에이전트입니다. 주요 기능은 다음과 같이 분류할 수 있습니다.

### 1.1. 자율적 작업 수행 및 자동화
Manus는 내장된 샌드박스 환경(Ubuntu 22.04)에서 셸 명령어 실행, 파일 조작, 코드 작성 및 실행(Python, Node.js)을 자율적으로 수행할 수 있습니다. 이를 통해 복잡한 다단계 작업을 사람의 개입 없이 처리합니다.

| 기능 | 설명 | 활용 예시 |
| :--- | :--- | :--- |
| **예약된 작업 (Scheduled Tasks)** | Cron 표현식 또는 간격(Interval)을 기반으로 작업을 스케줄링합니다. 최소 5분 간격으로 반복 실행이 가능합니다. | 정기적인 데이터 수집, 일일 보고서 생성, 시스템 상태 모니터링 |
| **Wide Research** | 최대 2,000개의 병렬 서브태스크를 생성하여 광범위한 데이터를 동시에 수집하고 분석합니다. | 대규모 시장 데이터 수집, 다수 종목의 재무제표 동시 분석 |
| **클라우드 브라우저** | Manus 전용 브라우저를 통해 웹사이트 탐색, 로그인, 데이터 추출, 양식 작성 등을 사람처럼 수행합니다. | API가 없는 웹사이트의 데이터 크롤링, 웹 기반 관리자 페이지 제어 |

### 1.2. 데이터 분석 및 시각화
원시 데이터(CSV, Excel 등)를 업로드하면, Manus가 이를 분석하여 다양한 형태의 시각적 결과물로 변환합니다.

*   **지원 차트**: 선형, 막대, 원형, 산점도, 레이더, 히트맵 등
*   **출력 형식**: 슬라이드 덱(프레젠테이션), 대화형 대시보드, 상세 보고서, 독립형 웹페이지
*   **내장 라이브러리**: Pandas, Matplotlib, Plotly, Seaborn 등이 샌드박스에 사전 설치되어 있어 복잡한 데이터 처리 및 시각화 코드를 직접 작성하고 실행할 수 있습니다.

### 1.3. 멀티미디어 처리 및 문서화
텍스트, 이미지, 오디오, 비디오 등 다양한 형태의 미디어를 처리하고 상호 변환할 수 있습니다.

| 기능 | 설명 |
| :--- | :--- |
| **이미지 생성 및 이해** | 텍스트 설명으로 이미지를 생성하거나, 업로드된 이미지에서 텍스트 추출 및 내용을 분석합니다. |
| **비디오 및 오디오 분석** | 비디오 콘텐츠를 분석하여 요약하거나, 오디오 파일을 텍스트로 변환(STT)합니다. |
| **Manus 슬라이드** | 분석 결과나 조사 내용을 바탕으로 전문적인 프레젠테이션(HTML 기반 대화형 또는 이미지 기반)을 자동 생성합니다. |

### 1.4. 외부 시스템 통합 (Integrations)
Manus는 다양한 외부 도구 및 서비스와 원활하게 연결되어 워크플로의 중심 역할을 수행합니다.

*   **MCP 연결기 (Model Context Protocol)**: GitHub, Notion, Slack, Google Workspace 등 주요 서비스와 사전 구축된 통합을 제공합니다.
*   **사용자 지정 MCP 서버**: 조직 내부의 독점 시스템이나 자체 구축한 API와 연동할 수 있습니다.
*   **Mail Manus**: 고유한 봇 이메일 주소(`@mail.manus.ai`)를 통해 이메일 수신 시 자동으로 작업을 트리거하고 처리 결과를 회신합니다.

## 2. autotrade-core 프로젝트 맞춤 활용 방안

현재 진행 중인 `autotrade-core` 프로젝트(Phase 3: 전략 엔진 개발 예정)에 Manus의 기능들을 다음과 같이 적용하여 시스템의 고도화 및 운영 효율성을 극대화할 수 있습니다.

### 2.1. 예약된 작업(Scheduled Tasks)을 통한 정기 모니터링 및 보고
현재 텔레그램 봇(아사, 알림봇)을 통해 알림을 받고 있으나, Manus의 스케줄링 기능을 활용하면 정기적인 심층 분석 보고서를 받아볼 수 있습니다.

*   **일일 시장 요약 보고서**: 매일 장 마감 후, Manus가 KIS API를 통해 수집된 OHLCV 데이터를 분석하고, 주요 지표 변화 및 전략 시그널 발생 내역을 정리하여 텔레그램이나 GitHub Issue로 자동 보고하도록 설정할 수 있습니다.
*   **시스템 상태 점검**: 주기적으로 서버(WSL Ubuntu)의 상태 로그를 확인하고 이상 징후 발생 시 즉각적인 조치를 취하는 워크플로를 구성할 수 있습니다.

### 2.2. Wide Research를 활용한 백테스팅 데이터 수집 및 분석
Phase 3(전략 엔진 개발) 및 Phase 4(백테스팅 프레임워크) 진행 시 대량의 과거 데이터 처리가 필수적입니다.

*   **병렬 데이터 처리**: 수십~수백 개 종목의 과거 데이터를 동시에 수집하고 전처리하는 작업에 Wide Research(병렬 처리) 기능을 적용하여 소요 시간을 획기적으로 단축할 수 있습니다.
*   **다중 전략 동시 검증**: 여러 가지 매매 전략 파라미터 조합을 병렬 서브태스크로 분배하여 동시에 백테스트를 수행하고 결과를 종합할 수 있습니다.

### 2.3. 데이터 분석 및 시각화를 통한 전략 성과 평가
단순한 텍스트 로그나 수치 데이터만으로는 전략의 유효성을 직관적으로 파악하기 어렵습니다.

*   **대화형 대시보드 생성**: 백테스트 결과 데이터(수익률, MDD, 승률 등)를 Manus에 제공하여, 전략의 성과를 한눈에 파악할 수 있는 대화형 대시보드나 상세 분석 보고서를 자동 생성하도록 지시할 수 있습니다.
*   **시각적 리포트**: 매매 타점(진입/청산 지점)이 표시된 가격 차트를 Matplotlib이나 Plotly를 사용하여 생성하고, 이를 문서화하여 전략 개선의 근거 자료로 활용할 수 있습니다.

### 2.4. 사용자 지정 MCP 서버를 통한 아사(OpenClaw)와의 직접 연동
현재는 Commander가 텔레그램을 통해 아사에게 명령을 내리고, 사자(Manus)는 GitHub을 통해 간접적으로 협업하는 구조입니다.

*   **직접 통신 채널 구축**: 아사(OpenClaw) 시스템에 간단한 MCP 서버 인터페이스를 구현하면, 사자(Manus)가 텔레그램을 거치지 않고도 직접 아사의 상태를 조회하거나 특정 스크립트 실행을 지시할 수 있습니다. 이를 통해 "사자(계획/분석) → 아사(실행) → 사자(결과 검증)"의 완전 자동화된 루프를 구축할 수 있습니다.

### 2.5. Mail Manus를 활용한 비상 알림 및 제어
텔레그램 외에 추가적인 제어 채널로 이메일을 활용할 수 있습니다.

*   **이메일 기반 긴급 제어**: 시스템에 심각한 오류가 발생하거나 급격한 시장 변동 시, Commander가 특정 키워드가 포함된 이메일을 Mail Manus 주소로 보내면, Manus가 이를 인식하여 즉시 모든 매매를 중지(Kill Switch)하는 등의 긴급 조치를 수행하도록 워크플로를 구성할 수 있습니다.

## 3. 결론 및 다음 단계 제안

Manus는 단순한 코드 작성 보조 도구를 넘어, 데이터 수집, 분석, 시각화, 스케줄링, 외부 시스템 연동을 아우르는 강력한 자동화 플랫폼입니다. 

**Commander께 드리는 제안:**
1.  **Phase 3 (전략 엔진 개발) 진행 시**: 전략의 성과를 시각적으로 검증하기 위해 Manus의 데이터 분석 및 시각화 기능을 적극 활용해 볼 것을 제안합니다.
2.  **정기 보고 자동화**: 예약된 작업(Scheduled Tasks) 기능을 사용하여 매일 아침 전일 장 마감 기준의 요약 브리핑을 생성하는 워크플로를 우선적으로 테스트해 보는 것이 좋겠습니다.
